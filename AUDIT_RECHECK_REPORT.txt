Summary:

- The test suite is green, but several fragile installer and team-sync paths are still not covered by automated tests.

Findings:

1. HIGH - update can mark stale config migrations as current without running them
Files:
- src/commands/update.ts:638-644
- src/commands/update.ts:745-758
- src/lib/version.ts:224-241
Evidence:
- update uses binary VERSION as localVersion and calculates comparison from latestVersion vs VERSION.
- runMigrations(latestVersion) only runs when binary comparison > 0.
- updateVersion(latestVersion) always runs afterward, even if version.json was stale and migrations were skipped.
Impact:
- A user can have the latest binary but an old version.json. Running update will skip migrations, then write version.json to latest, permanently masking pending migrations.
Suggested fix:
- Read version.json separately and run migrations when versionInfo.version < latestVersion, independent of binary VERSION. Only update version.json after migrations succeed.

2. HIGH - Windows installer continues after opencode-jce CLI install failure
File:
- install.ps1:430-500
Evidence:
- The CLI install block throws internally on dependency/copy failures, but the outer catch only Write-Warn's at line 499 and continues.
- Final summary still prints installed/manage instructions at install.ps1:918-960.
Impact:
- Windows install can appear successful while opencode-jce is missing or broken.
Suggested fix:
- Treat opencode-jce CLI install failure as fatal, or track failed install state and fail before final summary.

3. HIGH - offline installer overwrites existing config instead of safe-merge
File:
- scripts/bundle-offline.sh:30-36
Evidence:
- Generated install-offline.sh directly copies agents.json, mcp.json, lsp.json, and profiles into the user's config path.
Impact:
- Offline install can overwrite user-customized agents, MCP, LSP, and profiles, violating the safe-merge convention used elsewhere.
Suggested fix:
- Bundle and call scripts/merge-config.ts from install-offline.sh when Bun is available. Fallback should copy only missing files or back up before overwrite.

4. HIGH - offline installer does not create or merge opencode.json
File:
- scripts/bundle-offline.sh:30-37
Evidence:
- Offline installer copies legacy mcp.json and lsp.json but never creates/merges opencode.json.
- OpenCode native config uses opencode.json mcp/lsp keys.
Impact:
- Offline install can finish without registering context-keeper or other MCP servers in the config OpenCode actually reads.
Suggested fix:
- Use merge-config.ts in offline install, or directly create/merge opencode.json with buildDefaultMcpConfig and detected LSPs.

5. HIGH - existing old context-keeper MCP entries are not repaired
Files:
- scripts/merge-config.ts:383-400
- install.sh:416-418
- install.ps1:673-678
- src/lib/version.ts:126-139
Evidence:
- Merge/install paths add missing MCP defaults only when the key is absent.
- The historical migration exits early if config.mcp["context-keeper"] exists.
- Old context-keeper entries can lack env.PROJECT_ROOT.
Impact:
- Users who already have a partial context-keeper entry keep the broken entry and may read/write .opencode-context.md from the MCP server cwd instead of project root.
Suggested fix:
- Treat context-keeper as a managed entry and patch required fields when present: type, command, env.PROJECT_ROOT, enabled.

6. HIGH - credential-bearing GitHub URLs can be logged and persisted
Files:
- src/commands/plugin.ts:13-17
- src/lib/plugins.ts:227-249
- src/lib/plugins.ts:173-180
- src/commands/team.ts:14-20
- src/lib/team.ts:52-64
Evidence:
- parseGitHubPluginUrl rejects protocol/host/path issues but does not reject parsed username/password.
- plugin install logs the full githubUrl and stores source: githubUrl.
- team init logs and stores repoUrl directly.
Impact:
- URLs like https://TOKEN@github.com/org/repo can leak secrets into terminal output, logs, plugins.json, or team.json.
Suggested fix:
- Reject URLs with username/password. Normalize persisted GitHub URLs to https://github.com/owner/repo. Redact URL values before logging.

7. HIGH - setup writes source-able shell file with unescaped secrets
File:
- src/commands/setup.ts:114-136
Evidence:
- Raw API keys are inserted into export lines: export OPENAI_API_KEY="${openai}".
- The same file is recommended for shell sourcing.
Impact:
- Quotes/newlines/shell metacharacters in input can break the file or execute unintended shell syntax when sourced. Keys are also typed through visible readline input.
Suggested fix:
- Use hidden input and shell-safe single-quote escaping, or avoid shell source files and store only variable names. If writing secrets, use atomic write and restrictive permissions/ACLs.

8. HIGH - team pull overwrites local executable/config behavior without validation or backup
File:
- src/lib/team.ts:203-225
Evidence:
- pullTeamConfig copies agents.json, mcp.json, lsp.json, and profile JSON from the repo directly into the live config directory.
Impact:
- A compromised or mistaken team repo can replace agent prompts, MCP definitions, LSP commands, and profiles. Invalid JSON can break commands.
Suggested fix:
- Stage pulled config, validate schemas, show summary/diff, back up current files, and require explicit confirmation for destructive writes.

9. MEDIUM - team push may reuse a stale .team-sync repository after clone failure
File:
- src/lib/team.ts:83-104
Evidence:
- pushTeamConfig attempts git clone into .team-sync. If clone fails and tempDir exists, it runs git pull inside that directory without verifying remote URL/branch.
Impact:
- A stale or tampered .team-sync directory can cause push to the wrong repository.
Suggested fix:
- Remove tempDir before clone, or verify git remote get-url origin and branch before reuse.

10. MEDIUM - PowerShell LSP merge reports success without checking exit code
File:
- install.ps1:899-912
Evidence:
- bun run ... setup --merge-lsp output is captured, but $LASTEXITCODE is never checked.
- PowerShell external command failures do not throw by default under this pattern.
Impact:
- Installer can print "LSP servers merged into opencode.json" even if merge failed.
Suggested fix:
- Capture $LASTEXITCODE after line 907 and warn/fail if non-zero, including command output.

11. MEDIUM - existing opencode.json merge adds MCP only, not plugin or LSP defaults
File:
- scripts/merge-config.ts:383-400
Evidence:
- For existing opencode.json, ensureOpenCodeJson only initializes/merges config.mcp.
- Fresh opencode.json gets plugin and detected lsp at lines 412-419, but existing configs do not.
Impact:
- Existing users can miss the superpowers plugin registration and auto-detected LSP entries after install/update.
Suggested fix:
- Merge missing plugin array entries and non-conflicting detected LSP entries for existing opencode.json.

12. MEDIUM - LSP extension mapping is duplicated and out of sync
Files:
- src/lib/opencode-json-template.ts:17-56
- src/lib/config.ts:151-197
- scripts/merge-config.ts:297-340
Evidence:
- opencode-json-template.ts lacks mappings present elsewhere: zsh, yml, htm, less, hcl, sbt, jsonc.
Impact:
- Fresh opencode.json auto-detection misses valid file extensions for some LSPs.
Suggested fix:
- Move filetype-to-extension mapping to one shared module and import it from all paths.

13. MEDIUM - fallback schema disagrees with runtime validation
Files:
- schemas/fallback.schema.json:30-45
- src/lib/fallback.ts:36-67
Evidence:
- Schema allows any http(s):// endpoint and timeoutMs minimum 0.
- Runtime only accepts trusted endpoints and rejects timeoutMs <= 0.
Impact:
- opencode-jce validate can pass a fallback config that fallback status/test rejects.
Suggested fix:
- Align schema with runtime or invoke runtime fallback validation during validate.

14. MEDIUM - Anthropic fallback health check likely uses wrong protocol
Files:
- src/lib/fallback.ts:26-29
- src/lib/fallback.ts:149-164
Evidence:
- Anthropic endpoint is /v1/messages, but health check sends HEAD with Authorization: Bearer.
Impact:
- Valid Anthropic credentials may be reported unhealthy/degraded if Anthropic requires provider-specific headers/methods.
Suggested fix:
- Implement provider-specific checks: Anthropic header/method, OpenAI models endpoint, Ollama local unauthenticated check.

15. MEDIUM - corrupted memory JSON is silently discarded and can be overwritten
File:
- src/lib/memory.ts:70-90
Evidence:
- loadFile catches all errors and returns []. saveFile then writes a fresh file.
Impact:
- A malformed or temporarily unreadable memory file can be silently wiped on next write.
Suggested fix:
- Fail loudly on parse errors, create .bak before writes, and write atomically via temp file + rename.

16. MEDIUM - profile schema permits configs runtime assumes are complete
Files:
- schemas/profile.schema.json:5,24-35
- src/commands/use.ts:40-42,85-92
Evidence:
- Schema required omits model, maxTokens, and temperature.
- use command prints those fields unconditionally.
Impact:
- validate can pass profiles that later display undefined or behave incorrectly.
Suggested fix:
- Add model, maxTokens, and temperature to required fields, or make command code handle variant profile shapes explicitly.

17. MEDIUM - invalid dashboard/optimize period values silently fall back
Files:
- src/commands/dashboard.ts:143-158
- src/commands/optimize.ts:32-45
Evidence:
- Unknown dashboard period falls to month; unknown optimize period falls to week.
Impact:
- Typos like --period yearly produce misleading analytics instead of an actionable error.
Suggested fix:
- Use Commander choices or explicit validation for allowed periods.

18. MEDIUM - route command persists prompt excerpts to logs
Files:
- src/commands/route.ts:12-13
- src/lib/logger.ts:78-80
Evidence:
- route logs the first 50 characters of the prompt as JSON args.
Impact:
- Secrets, PII, or proprietary code included in prompts can be persisted to opencode-jce.log.
Suggested fix:
- Log prompt length/hash only, or redact prompt content unless debug logging is explicitly enabled.

19. MEDIUM - integration tests are weak for CLI behavior
File:
- tests/integration/cli.test.ts:30-83
Evidence:
- Several command tests accept exit 0 or 1 and assert only that commands do not crash.
Impact:
- User-facing regressions can pass unnoticed.
Suggested fix:
- Isolate config with temp env vars and assert meaningful stdout/stderr/exit behavior per command.

20. LOW - setup next-step message uses literal ${envPath}
File:
- src/commands/setup.ts:135-136
Evidence:
- info uses a normal double-quoted string instead of a template literal.
Impact:
- User sees source ${envPath}, not the actual path.
Suggested fix:
- Use a template literal: info(`Add \`source ${envPath}\` ...`).

21. LOW - context archive marker drops part of the original editable notice
File:
- src/mcp/context-keeper.ts:221-225
Evidence:
- Replacement targets only "> Auto-maintained by AI." and replaces it with a shorter archive notice line.
Impact:
- The original "You can edit this file freely" text can be lost during archive marker insertion.
Suggested fix:

22. LOW - installer summary counts are stale
File:
- install.ps1:942-946
Evidence:
- Summary says 20 Profiles and 6 MCP Tools.
- Current tests/config indicate 19 profiles and OpenCode-native MCP defaults contain 9 servers.
Impact:
- User-facing install output is inaccurate.
Suggested fix:

23. LOW - agent IDs are not validated despite prompt/schema expectations
Files:
- src/commands/agent.ts:33-46,67-77
- schemas/agents.schema.json:14-17
Evidence:
- Prompt says lowercase/no spaces, but schema only requires a non-empty string and create command does not validate.
Impact:
- Agent IDs with spaces/case/path-like text can create confusing references.
Suggested fix:
- Enforce a shared regex such as ^[a-z0-9][a-z0-9-]*$ in both command and schema.
