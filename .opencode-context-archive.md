## Archived: 2026-05-05
### Architecture Decisions
- CLI installed to `~/.config/opencode/cli/` via .cmd shim in bun bin
- Config deployed to `~/.config/opencode/` on all platforms, including Windows
- Installer never overwrites existing user config (safe merge)
- Context preservation enforced via MCP server `context-keeper` (src/mcp/context-keeper.ts)
- context-keeper auto-registered to `opencode.json` (NOT mcp.json) via installer + migration
- `config/mcp.json` = Claude Desktop format (legacy); OpenCode reads `opencode.json` key `"mcp"`
- Backup enforcement: TodoWrite completion = context update (IRON RULE in AGENTS.md)
- Auto-Dispatch in AGENTS.md: AI silently selects agent + loads skills per message
- `opencode-jce update` backs up AGENTS.md before overwriting
- LSP auto-detect: installer scans PATH for all lsp.json commands, pre-populates opencode.json
- context-keeper responses include REMINDER lines to enforce AI compliance
- loadOpenCodeConfig() auto-creates full template if file missing (never produces partial config)
- install.ps1 uses [System.IO.File]::WriteAllText with UTF8NoBOM (PS 5.1 compat)
- `opencode-jce update` now also updates ~/.config/opencode/cli/ folder (not just global binary)
- `opencode-jce update` always syncs files (no early exit on same version); --force deprecated

## Archived: 2026-05-06
### Architecture Decisions
- Sisyphus CLI exports testable clear and trace-limit helpers; clear uses deterministic timestamp backup names when given now.
- Sisyphus CLI uses createSisyphusCommand injection to behavior-test commands without process.exit.
- Sisyphus v2 Phase 6 CLI exposes status, trace, report, and clear over persisted execution memory.
- Recovery retry budget is consumed only when an actual retry task is created, not when a retryable failure is recorded.
- Real recovery loop retries failed/invalid delegated tasks through collect-time recovery and blocks with handoff when budget or category prevents retry.
- Recovery retry tasks are launched through spawner after manager creates retry lineage; spawner supports prompt, promptAsync, and chat SDK methods.
- SDK prompt calls use `{ path: { id }, body: { agent, parts: [{ type: "text", text }] } }`; legacy chat fallback keeps `{ params, body.content }`.
- Final review gate evaluates workflow completion, certificate validity, blockers, delegated review acceptance, and exhausted retry history before pass/block.
- Final review gate blocks unresolved structured retry exhaustion via status text, retry flags, or retryCount >= maxRetries; resolved records do not block.
- Final review gate requires retry counts at max to also have an unresolved failure/blocker marker before blocking; delegated review acceptance uses explicit positive formats only.
- Final review gate blocks completion on missing verification, active blockers, unaccepted delegated work, unresolved exhausted retries, or invalid certificate.
- Dispatch prompts now use a formal delegation envelope with goal, scope, non-goals, constraints, allowed files, expected verification, timeout hint, and output contract.
- Plugin completion guard invokes final review gate for completion-like tool output when persisted activeWorkflow exists.
- Blocked final gates append FINAL REVIEW GATE warning before generic verification warning.
- Completed delegated task memory now preserves review status/notes and verification summary so final review gates and CLI reports can reason over accepted/blocked delegated work.
- Final review gate production wiring derives delegated review strings from persisted completed summaries and verification evidence.
- Blocked delegated collect results are converted to recovery blockers instead of being reported as completed success.
- Execution memory merge preserves persisted task history collections by id while refreshing current manager snapshots and workflow runtime.
- Plugin bg_collect persists manager memory immediately so final review gate sees accepted/blocked delegated results without waiting for idle events.
- Accepted retry lineage is marked resolved so successful retries do not trip exhausted retry final-gate checks.
- Plugin runtime agent IDs rebranded to jce-worker and jce-researcher; legacy settings map sisyphus/librarian to new IDs.
- Full JCE-Worker migration removes opencode-jce sisyphus command; legacy files/settings are migration-only inputs.
- Workflow cloneRun deep-copies route skills, evidence entries, and blocker arrays to preserve prior snapshot immutability.
- JCE-Worker status/report expose active workflow routing metadata for operator observability.
