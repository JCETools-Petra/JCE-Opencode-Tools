Findings (deduplicated, ordered by severity):

1. CRITICAL - Anthropic health endpoint URL is misspelled everywhere
Files:
- src/lib/fallback.ts:27
- src/lib/fallback.ts:108
- config/fallback.json:6
Evidence:
- Domain is api.anxthxropic.com (with extra x characters) instead of api.anthropic.com.
- TRUSTED_PROVIDER_ENDPOINTS allowlist uses this same misspelled domain.
- Default fallback config and shipped config/fallback.json also use it.
Impact:
- Anthropic health checks always fail (DNS resolution error or wrong host).
- Fallback system never selects Anthropic as healthy provider.
- If the misspelled domain is registered by a malicious actor, x-api-key header is sent to it.
Suggested fix:
- Replace api.anxthxropic.com with api.anthropic.com in all three locations.

2. HIGH - Migration (1.4.0 to 1.6.0) registers context-keeper without env.PROJECT_ROOT
File:
- src/lib/version.ts:135-138
Evidence:
- Migration creates context-keeper entry with only type, command, enabled.
- Missing env: { PROJECT_ROOT: "${PROJECT_ROOT}" } which is present in all other registration paths.
- The repair code in update.ts ensureOpenCodeJson fixes this, but only when update is run.
Impact:
- Users who triggered this migration but never ran update will have context-keeper writing .opencode-context.md to process.cwd() instead of project root.
Suggested fix:
- Add env: { PROJECT_ROOT: "${PROJECT_ROOT}" } to the migration's context-keeper config object.

3. HIGH - Archive note injection corrupts header line via substring replace
File:
- src/mcp/context-keeper.ts:221-225
Evidence:
- Replace target is "> Auto-maintained by AI." but actual template line is "> Auto-maintained by AI. You can edit this file freely."
- String.replace matches the substring and inserts archive note in the MIDDLE of the line.
- Result: "> Auto-maintained by AI.\n> Archived entries: see ... You can edit this file freely."
Impact:
- After first archive operation, context file header becomes malformed.
Suggested fix:
- Use a full-line regex: updated.replace(/^> Auto-maintained by AI\..*$/m, "> Auto-maintained by AI. You can edit this file freely.\n> Archived entries: see .opencode-context-archive.md")

4. HIGH - team push/pull passes unsanitized repoUrl and branch to git clone
File:
- src/lib/team.ts:91, 188
Evidence:
- teamConfig.repoUrl from user-controlled team.json is passed directly to Bun.spawn git clone.
- teamConfig.branch is also passed directly to --branch.
- sanitizeGitUrl strips credentials but does not validate protocol or prevent local path injection.
Impact:
- Tampered team.json can cause clone from arbitrary locations or execute commands via git transport.
Suggested fix:
- Validate repoUrl is HTTPS URL to a known git host. Validate branch against ^[a-zA-Z0-9._/-]+$.

5. MEDIUM - install.sh summary reports wrong counts (20 Profiles, 8 MCP Servers)
File:
- install.sh:812-813
Evidence:
- Line 812: "20 Profiles" but only 19 exist.
- Line 813: "8 MCP Servers" but 9 are configured.
- install.ps1 correctly says 19 Profiles and 9 MCP Tools.
Impact:
- Cosmetic inconsistency in installer output.
Suggested fix:
- Change to 19 Profiles and 9 MCP Servers.

6. MEDIUM - fallback.schema.json does not require maxRetries and timeoutMs
File:
- schemas/fallback.schema.json:5
Evidence:
- Schema required array only has "providers".
- Runtime validateFallbackConfig() requires both maxRetries and timeoutMs.
Impact:
- validate command can pass a fallback config that fallback status/test rejects at runtime.
Suggested fix:
- Add "maxRetries" and "timeoutMs" to the required array.

7. MEDIUM - profile.schema.json does not require model/maxTokens for non-routing profiles
Files:
- schemas/profile.schema.json:5
- src/commands/use.ts:85-92
Evidence:
- Schema required omits model and maxTokens.
- use command prints profile.model and profile.maxTokens unconditionally.
- hybrid-hemat (provider: auto) legitimately lacks these, but all other profiles have them.
Impact:
- A non-routing profile without model/maxTokens passes validate but shows undefined in CLI.
Suggested fix:
- Use a conditional schema (if/then/else or oneOf) that requires model+maxTokens when provider is not "auto".

8. MEDIUM - team init does not validate git URL format
File:
- src/lib/team.ts:53-68
Evidence:
- initTeamSync accepts any string as repoUrl.
- sanitizeGitUrl only strips credentials but does not validate protocol/host.
Impact:
- Invalid URLs are saved to team.json and only fail later on push/pull with cryptic git errors.
Suggested fix:
- Validate URL is parseable and uses https: or git: protocol before saving.

9. MEDIUM - prompt template apply/reset uses first double-newline as delimiter
Files:
- src/lib/prompts.ts:73-85
- src/lib/prompts.ts:99-104
Evidence:
- applyPromptToAgent finds first \n\n to determine where old template ends.
- resetAgentPrompt regex matches up to first \n\n.
- Multi-paragraph templates contain \n\n internally.
Impact:
- Applying or resetting multi-paragraph templates corrupts agent system prompts.
Suggested fix:
- Use a unique end-marker like [/template] or store template boundaries separately.

10. MEDIUM - replaceSection drops blank separator lines between sections
File:
- src/mcp/context-keeper.ts:144-182
Evidence:
- When replacing section content, all lines between heading and next ## are dropped.
- The blank line that separates sections is lost.
Impact:
- Over multiple updates, .opencode-context.md becomes compressed without visual separation.
Suggested fix:
- Add a blank line before the next section heading when exiting a replaced section.

11. MEDIUM - fixer.ts splits install commands naively on spaces
File:
- src/lib/fixer.ts:228-229
Evidence:
- server.installCommand.split(" ") breaks if paths contain spaces.
Impact:
- Install commands with spaces in paths fail silently.
Suggested fix:
- Use shell execution or document that installCommand must not contain spaces in paths.

12. MEDIUM - tokens.ts assumes time_created is milliseconds without validation
File:
- src/lib/tokens.ts:261-286
Evidence:
- Passes Date.getTime() (ms) to WHERE clause and constructs new Date(row.time_created).
- If OpenCode stores seconds, all time-filtered queries return empty and dates show 1970.
Impact:
- Potential silent data loss if OpenCode changes timestamp format.
Suggested fix:
- Add sanity check: if time_created < 1e12, multiply by 1000.
